### Docker Desktop 4.65 Inference Manager bug

跑 Docker Desktop 4.65.0 在 Windows 上,backend 初始化 Inference Manager 時 bind Unix socket 失敗:

```
listening on unix://C:\Users\yen\AppData\Local\Docker\run\dockerInference:
bind: An attempt was made to access a socket in a way forbidden
listener: The filename, directory name, or volume label syntax is incorrect
```

這是 Docker Desktop 4.40+ 加 Model Runner 後的 Windows 已知 bug。settings.json 試過加 `ModelRunnerEnabled=false` 等 keys 都無效。

**解決方案**：降版到 Docker Desktop 4.39.0（前一版,沒 Model Runner）。
